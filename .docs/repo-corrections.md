- In Dusk assertions for Blade table rows, avoid exact `textContent` equality because whitespace/line breaks are unstable; normalize whitespace and assert invariant fragments plus `data-*` selectors and semantic classes (for example green/red validation states).
- In admin user creation flows that email an access code, set both `code` and initial `password` to the same generated value in the creation action; sending one credential while persisting another causes immediate login failures in production. Also keep mail assertions focused on forbidden CTA/link fragments (e.g., password-change route/token) instead of banning all `href` tags, because templates may include legitimate base links.
- For new admin download endpoints, always add paired Feature tests: one `assertOk()` for the allowed admin role and one `assertForbidden()` for a lower-privilege role. This catches access-regression bugs early and documents intended authorization.
- In this repo, `User::assignRole(...)` silently does nothing if the role row is missing; in role-based Feature tests, seed roles first (`Role::firstOrCreate`) or use helpers that guarantee role existence to avoid false 403 failures.
- If Docker app containers lack `mysqldump`/`mariadb-dump`, provide a DB-backed SQL streaming fallback for admin backups so the feature works without extra OS packages; keep CLI dump as the preferred fast path when available.
- In `DevSeeder` vote generation, keep idempotency by checking existing `VotingBallot` (`voting_id` + `owner_id`) immediately before cast and gracefully skipping `already_voted` ValidationException from `CastVotingBallotAction`; this prevents `db:seed` re-runs from crashing.
- For download endpoints that return physical files (like sqlite backups), set an explicit `Content-Type` in the response and assert it in Feature tests; temp-file MIME guessing can return `text/plain` and break deterministic header expectations.
- In Livewire table tests, avoid asserting removed/static labels (e.g., `%Erakina`, `%Komunidad`) when the UI now renders only formatted values; assert stable data values/selectors instead to prevent false failures from presentation copy refactors.
- If CI executes Feature/Livewire tests without APP_KEY, set a deterministic fallback app key in `tests/TestCase::setUp()` (and sync to env/server vars) to prevent `MissingAppKeyException` during view rendering.
